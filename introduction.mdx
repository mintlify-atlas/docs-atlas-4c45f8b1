Flamegraph brings Python performance profiling into VS Code without leaving your editor. Profile Python scripts and Jupyter notebooks using [py-spy](https://github.com/benfred/py-spy) for CPU profiling or [memray](https://github.com/bloomberg/memray) for memory profiling, then explore results in two linked views: an interactive flamegraph panel and inline annotations displayed alongside your code.

## How it works

After profiling completes, results appear in two linked views:

1. **Inline code annotations** — timing or memory information is displayed for each function scope in the editor, with colors indicating the call depth level.
2. **Interactive flamegraph panel** — the complete call stack is visualized as a flamegraph in a new tab. You can:
   - Click any frame to zoom in; click a parent frame to zoom out
   - Cmd+Click (Mac) or Ctrl+Click (Windows/Linux) to open the corresponding source file
   - Click legend entries to filter the flamegraph by module
   - Toggle between source file and function name display using the `</>` button in the legend

Selecting a frame in the flamegraph automatically filters the inline annotations to show only the profiling data visible in the current view.

## Prerequisites

- **VS Code** 1.95.0 or later
- **Python extension** (`ms-python.python`) — required
- **Jupyter extension** (`ms-toolsai.jupyter`) — required for notebook profiling

## Profiler requirements

**py-spy** is used for CPU profiling and is automatically installed by the extension if not already present.

**memray** is used for memory profiling and must be installed manually. It is only supported on Linux, macOS, and within WSL or Docker containers — not on native Windows.

```bash
pip install memray
```

<Note>
On some Linux distributions and on macOS, py-spy requires elevated permissions to sample other processes. The extension will prompt you to use `sudo` when needed. You can also set `flamegraph.py-spy.alwaysUseSudo` to `true` in your VS Code settings to skip the prompt.
</Note>
