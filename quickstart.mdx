---
title: "Quickstart"
description: "Install the Flamegraph extension and profile your first Python script in minutes."
---

<Steps>
  <Step title="Install the extension">
    Open VS Code and install the Flamegraph extension from the Marketplace.

    1. Open the **Extensions** panel (`Ctrl+Shift+X` / `Cmd+Shift+X`)
    2. Search for **Flamegraph**
    3. Click **Install** on the extension by `rafaelha`

    Or install directly from the terminal:

    ```bash
    code --install-extension rafaelha.vscode-flamegraph
    ```

    <Note>
      The extension requires the [Python extension](https://marketplace.visualstudio.com/items?itemName=ms-python.python) and the [Jupyter extension](https://marketplace.visualstudio.com/items?itemName=ms-toolsai.jupyter). These will be installed automatically as dependencies.
    </Note>
  </Step>

  <Step title="Open a Python file">
    Open any Python script (`.py` file) in VS Code. Make sure you have a Python interpreter selected — the extension uses your active Python environment to find and run py-spy.

    <Tip>
      If py-spy is not installed, the extension will prompt you to install it automatically on first use.
    </Tip>
  </Step>

  <Step title="Start profiling">
    There are three ways to start profiling a Python script:

    **Option A — Run button dropdown:**
    Click the dropdown arrow next to the ▶️ run button in the editor title bar and select **Flamegraph: Profile file with py-spy**.

    **Option B — Command Palette:**
    Open the Command Palette (`Ctrl+Shift+P` / `Cmd+Shift+P`) and run:
    ```
    Flamegraph: Profile file with py-spy
    ```

    **Option C — Explorer context menu:**
    Right-click a `.py` file in the Explorer panel and select **Flamegraph: Profile file with py-spy**.

    The profiler runs in a VS Code task terminal. You can stop it at any time with `Ctrl+C`.
  </Step>

  <Step title="Inspect results">
    When profiling completes, two views appear simultaneously:

    **Flamegraph panel** — opens in a new tab showing the full call stack. Each bar represents a function; width represents time spent. Colors identify Python modules.

    **Inline annotations** — timing information appears as decorations next to each function in your source files.

    The two views are linked: clicking a node in the flamegraph filters the inline annotations to show only timing data for that portion of the call stack.
  </Step>

  <Step title="Navigate the flamegraph">
    Use these interactions to explore the flamegraph:

    | Action | Result |
    |--------|--------|
    | Click a frame | Zoom in to that frame |
    | Click a parent frame | Zoom out |
    | `Cmd+Click` (Mac) / `Ctrl+Click` (Windows/Linux) | Jump to source file at that line |
    | Click `all` node | Opens the profiled file in the editor |
    | Click a legend item | Toggle visibility of that module |
    | Type in the filter box | Filter frames by filename, function name, or module |
    | Click `</>` in legend | Toggle between code and function name display |
  </Step>
</Steps>

## Toggle inline annotations

To hide or show the inline timing decorations, right-click anywhere in the editor and select **Flamegraph: Toggle Inline Profile**, or use the Command Palette.

<Tip>
  The command `Flamegraph: Show` reopens the flamegraph panel if you close it. The panel remembers the last selected frame.
</Tip>
